# データモデル: 注文機能

## エンティティ

### 注文（Order）

| フィールド | 型 | 制約 | 説明 |
|-----------|------|------|------|
| id | UUID | 必須、一意 | 注文ID |
| userId | UUID | 必須 | 注文者のユーザーID |
| items | OrderItem[] | 必須、1件以上 | 注文明細のリスト |
| totalAmount | 整数 | 必須、0以上 | 注文合計金額（税抜） |
| status | OrderStatus | 必須 | 注文ステータス |
| createdAt | 日時 | 必須、自動生成 | 注文日時 |
| updatedAt | 日時 | 必須、自動更新 | 最終更新日時 |

### 注文明細（OrderItem）

| フィールド | 型 | 制約 | 説明 |
|-----------|------|------|------|
| productId | UUID | 必須 | 商品ID |
| productName | 文字列 | 必須 | 商品名（注文時点のスナップショット） |
| price | 整数 | 必須、0以上 | 単価（注文時点のスナップショット） |
| quantity | 整数 | 必須、1以上 | 数量 |

**備考**: 小計は `price × quantity` で算出される導出値であり、フィールドとして保持しない。

## ステータス遷移（ステートマシン）

### ステータス定義

| ステータス | 英名 | 説明 |
|-----------|------|------|
| 保留中 | pending | 注文作成直後の初期状態 |
| 確認済み | confirmed | 管理者が注文を確認した状態 |
| 発送済み | shipped | 商品を発送した状態 |
| 配送完了 | delivered | 商品が届いた最終状態 |
| キャンセル | cancelled | 注文がキャンセルされた最終状態 |

### 遷移ルール

```text
┌─────────┐    確認     ┌──────────┐    発送     ┌─────────┐    配送完了   ┌──────────┐
│ pending │──────────→│ confirmed│──────────→│ shipped │──────────→│ delivered│
│ (保留中) │            │ (確認済み) │            │ (発送済み)│            │ (配送完了) │
└────┬────┘            └────┬─────┘            └─────────┘            └──────────┘
     │                      │                                          最終状態
     │ キャンセル             │ キャンセル
     │                      │
     └──────┐  ┌────────────┘
            ↓  ↓
      ┌───────────┐
      │ cancelled │
      │ (キャンセル) │
      └───────────┘
        最終状態
```

### 遷移マトリクス

| 遷移元＼遷移先 | pending | confirmed | shipped | delivered | cancelled |
|--------------|---------|-----------|---------|-----------|-----------|
| **pending**  | -       | OK        | -       | -         | OK        |
| **confirmed**| -       | -         | OK      | -         | OK        |
| **shipped**  | -       | -         | -       | OK        | -         |
| **delivered**| -       | -         | -       | -         | -         |
| **cancelled**| -       | -         | -       | -         | -         |

## リレーションシップ

```text
Order 1 ──── * OrderItem    : 1つの注文は1件以上の注文明細を持つ
Order * ──── 1 User          : 1人のユーザーは複数の注文を持つ
OrderItem * ── 1 Product     : 注文明細は商品への参照を持つ（スナップショット）
```

## バリデーションルール

- 注文作成時: カートが空でないこと、confirmed フラグが true であること
- ステータス更新時: ValidStatusTransitions マップに基づく遷移チェック
- 注文一覧: ページ番号 >= 1、1ページあたり件数 1〜100（デフォルト20）
- 注文詳細: 購入者は自分の注文のみ閲覧可能
