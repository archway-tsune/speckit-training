# API コントラクト: 注文機能

## エンドポイント一覧

| メソッド | パス | 認証 | 説明 |
|---------|------|------|------|
| GET | `/api/orders` | buyer/admin | 注文一覧取得 |
| GET | `/api/orders/:id` | buyer/admin | 注文詳細取得 |
| POST | `/api/orders` | buyer | 注文作成 |
| PATCH | `/api/orders/:id` | admin | ステータス更新 |

---

## GET /api/orders - 注文一覧取得

**認可ルール**:
- buyer: 自分の注文のみ（userId 自動フィルタ）
- admin: 全注文（userId パラメータで任意フィルタ可能）

**クエリパラメータ**:

| パラメータ | 型 | 必須 | デフォルト | 説明 |
|-----------|------|------|----------|------|
| page | 整数 | いいえ | 1 | ページ番号（1始まり） |
| limit | 整数 | いいえ | 20 | 1ページあたりの件数（1〜100） |
| status | 文字列 | いいえ | - | ステータスフィルタ |
| userId | UUID | いいえ | - | ユーザーIDフィルタ（admin のみ） |

**レスポンス（200）**:
```json
{
  "success": true,
  "data": {
    "orders": [
      {
        "id": "uuid",
        "userId": "uuid",
        "items": [...],
        "totalAmount": 3000,
        "status": "pending",
        "createdAt": "2026-02-08T00:00:00.000Z",
        "updatedAt": "2026-02-08T00:00:00.000Z"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 50,
      "totalPages": 3
    }
  }
}
```

---

## GET /api/orders/:id - 注文詳細取得

**認可ルール**:
- buyer: 自分の注文のみ（他人の注文は 404）
- admin: 全注文

**パスパラメータ**:

| パラメータ | 型 | 説明 |
|-----------|------|------|
| id | UUID | 注文ID |

**レスポンス（200）**: Order オブジェクト
**レスポンス（404）**: 注文が見つからない

---

## POST /api/orders - 注文作成

**認可ルール**: buyer のみ

**リクエストボディ**:
```json
{
  "confirmed": true
}
```

**処理フロー**:
1. buyer 認可チェック
2. confirmed フラグのバリデーション
3. カート取得（userId から自動）
4. カートが空でないことを確認
5. カート内容から注文明細を作成
6. 注文を保存（status: "pending"、totalAmount: カートの subtotal）
7. カートをクリア
8. 作成された注文を返却

**レスポンス（200）**: 作成された Order オブジェクト
**レスポンス（400）**: カートが空、バリデーションエラー

---

## PATCH /api/orders/:id - ステータス更新

**認可ルール**: admin のみ

**リクエストボディ**:
```json
{
  "id": "uuid",
  "status": "confirmed"
}
```

**処理フロー**:
1. admin 認可チェック
2. 入力バリデーション
3. 注文取得
4. ステータス遷移ルールの検証
5. ステータス更新
6. 更新された注文を返却

**レスポンス（200）**: 更新された Order オブジェクト
**レスポンス（400）**: 無効なステータス遷移
**レスポンス（404）**: 注文が見つからない
