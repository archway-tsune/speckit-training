# クイックスタート: 注文機能

## 前提条件

- カタログ機能（001-catalog-listing）が実装済み
- カート管理機能（002-cart-management）が実装済み
- セッション認証が動作可能

## テストシナリオ

### シナリオ1: チェックアウトと注文確定（購入者）

1. buyer としてログインする
2. カタログから商品をカートに追加する
3. `/checkout` に遷移する
4. 商品一覧と合計金額が表示されることを確認する
5. 「注文を確定」ボタンをクリックする
6. 注文完了画面に注文IDとお礼メッセージが表示されることを確認する
7. カートページに遷移し、カートが空であることを確認する

### シナリオ2: 注文履歴閲覧（購入者）

1. buyer としてログインする（事前に注文を作成済み）
2. `/orders` に遷移する
3. 自分の注文一覧が表示されることを確認する
4. 注文をクリックして詳細画面を開く
5. 注文ID・日時・ステータス・商品一覧・合計金額が表示されることを確認する

### シナリオ3: 他人の注文アクセス拒否（購入者）

1. buyer-A としてログインし、注文を作成する
2. buyer-B としてログインする
3. buyer-A の注文IDで `/orders/{id}` にアクセスする
4. アクセスが拒否されることを確認する

### シナリオ4: 注文管理（管理者）

1. admin としてログインする
2. `/admin/orders` に遷移する
3. 全注文が一覧表示されることを確認する
4. ステータスフィルタで「保留中」を選択し、フィルタが動作することを確認する
5. 注文をクリックして詳細画面を開く
6. ステータスを「確認済み」に変更し、更新が反映されることを確認する

### シナリオ5: ステータス遷移ルール検証（管理者）

1. admin としてログインする
2. 「配送完了」の注文詳細を開く
3. ステータス変更の選択肢がないことを確認する
4. 「発送済み」の注文詳細を開く
5. ステータス変更の選択肢が「配送完了」のみであることを確認する

### シナリオ6: 認可チェック

1. buyer としてログインする
2. `/admin/orders` にアクセスする
3. アクセスが拒否されることを確認する
4. 未ログイン状態で `/checkout` にアクセスする
5. ログインページにリダイレクトされることを確認する
