# 機能仕様書: 注文機能

**フィーチャーブランチ**: `003-order-management`
**作成日**: 2026-02-08
**ステータス**: ドラフト
**入力**: 購入者がカート内の商品を確認して注文を確定し、注文履歴・注文詳細を閲覧できる機能。管理者は全注文の一覧閲覧とステータス更新ができる。

## ユーザーシナリオ & テスト

### ユーザーストーリー 1 - チェックアウトと注文確定（優先度: P1）

購入者がカート内の商品を確認し、注文を確定できる。チェックアウト画面でカート内の商品一覧（商品名・単価・数量・小計）と合計金額を表示する。「注文を確定」ボタンで注文を作成し、注文完了画面に注文IDとお礼メッセージを表示する。注文作成後、カートは自動的にクリアされる。

**この優先度の理由**: 注文機能の中核であり、これがなければ購入フローが成立しない。最小限の価値提供単位として、カートから注文を作成できることが最優先。

**独立テスト**: カートに商品を追加した状態でチェックアウト画面を開き、注文確定ボタンを押して注文完了画面が表示されることを確認できる。

**受入シナリオ**:

1. **前提** 購入者としてログイン済みで、カートに商品が入っている、**操作** チェックアウト画面を開く、**結果** カート内の商品一覧（商品名・単価・数量・小計）と合計金額が表示される
2. **前提** 購入者としてログイン済みで、チェックアウト画面に商品一覧が表示されている、**操作** 「注文を確定」ボタンをクリックする、**結果** 注文が作成され、注文完了画面に注文IDとお礼メッセージが表示される
3. **前提** 購入者としてログイン済みで、注文が正常に確定された、**操作** カートページを開く、**結果** カートが空になっている
4. **前提** 購入者としてログイン済みで、カートが空の状態、**操作** チェックアウト画面を開く、**結果** 「カートに商品がありません」メッセージが表示され、注文確定ボタンは非表示
5. **前提** 未ログイン状態、**操作** チェックアウト画面にアクセスする、**結果** ログインページにリダイレクトされ、ログイン後にチェックアウト画面に戻る

---

### ユーザーストーリー 2 - 注文履歴・注文詳細（優先度: P2）

購入者が自分の過去の注文を一覧で確認し、各注文の詳細を閲覧できる。注文一覧では注文ID先頭8桁・注文日・商品数・合計金額・ステータスを表示する。注文詳細画面では注文ID・注文日時・ステータス・商品一覧・合計金額を確認できる。

**この優先度の理由**: 注文確定後に購入者が注文状況を確認できることは、購入体験の完結に不可欠。注文確定（P1）の次に重要。

**独立テスト**: 注文を1件以上作成した状態で注文履歴ページを開き、注文一覧と詳細画面が正しく表示されることを確認できる。

**受入シナリオ**:

1. **前提** 購入者としてログイン済みで、過去に注文がある、**操作** 注文履歴ページを開く、**結果** 自分の注文が一覧表示される（注文ID先頭8桁・注文日・商品数・合計金額・ステータス）
2. **前提** 購入者としてログイン済みで、注文一覧に21件以上の注文がある、**操作** 注文履歴ページを開く、**結果** 20件/ページでページネーションが表示される
3. **前提** 購入者としてログイン済みで、注文一覧が表示されている、**操作** 注文の1つをクリックする、**結果** 注文詳細画面に注文ID・注文日時・ステータス・商品一覧・合計金額が表示される
4. **前提** 購入者としてログイン済み、**操作** 他の購入者の注文IDで注文詳細画面にアクセスする、**結果** アクセスが拒否される（404または権限エラー）
5. **前提** 購入者としてログイン済みで、注文が0件、**操作** 注文履歴ページを開く、**結果** 「注文がありません」メッセージが表示される

---

### ユーザーストーリー 3 - 管理者の注文一覧閲覧（優先度: P3）

管理者が全ユーザーの注文を一覧で確認でき、ステータスで絞り込める。注文管理画面では注文ID・注文者・注文日・商品数・合計金額・ステータスを表示する。

**この優先度の理由**: 管理者が注文状況を把握できることは運用上必須だが、購入者フロー（P1, P2）が先に動作する必要がある。

**独立テスト**: 管理者としてログインし、注文管理ページで全注文の一覧が表示され、ステータスフィルタが動作することを確認できる。

**受入シナリオ**:

1. **前提** 管理者としてログイン済みで、注文が存在する、**操作** 注文管理ページを開く、**結果** 全ユーザーの注文が一覧表示される
2. **前提** 管理者としてログイン済みで、注文管理ページが表示されている、**操作** ステータス「保留中」で絞り込む、**結果** 保留中の注文のみが表示される
3. **前提** 管理者としてログイン済みで、注文一覧が表示されている、**操作** 注文の1つをクリックする、**結果** 注文詳細画面に注文の全情報が表示される
4. **前提** 購入者（buyer）としてログイン済み、**操作** 管理者の注文管理ページにアクセスする、**結果** アクセスが拒否される（403エラーまたはリダイレクト）

---

### ユーザーストーリー 4 - 注文ステータス更新（優先度: P4）

管理者が注文のステータスを更新できる。ステータス遷移には厳格なルールがあり、許可された遷移のみ実行可能。

ステータスの種類: 保留中（pending）、確認済み（confirmed）、発送済み（shipped）、配送完了（delivered）、キャンセル（cancelled）

遷移ルール:
- 保留中 → 確認済み、キャンセル
- 確認済み → 発送済み、キャンセル
- 発送済み → 配送完了
- 配送完了 → 遷移不可（最終状態）
- キャンセル → 遷移不可（最終状態）

**この優先度の理由**: ステータス更新は注文管理の運用に必要だが、一覧閲覧（P3）が先に動作する必要がある。

**独立テスト**: 管理者として注文詳細画面からステータスを変更し、遷移ルールに従った更新が正しく反映されることを確認できる。

**受入シナリオ**:

1. **前提** 管理者としてログイン済みで、「保留中」の注文詳細画面を開いている、**操作** ステータスを「確認済み」に変更する、**結果** ステータスが「確認済み」に更新される
2. **前提** 管理者としてログイン済みで、「保留中」の注文詳細画面を開いている、**操作** ステータスを「キャンセル」に変更する、**結果** ステータスが「キャンセル」に更新される
3. **前提** 管理者としてログイン済みで、「確認済み」の注文詳細画面を開いている、**操作** ステータスを「発送済み」に変更する、**結果** ステータスが「発送済み」に更新される
4. **前提** 管理者としてログイン済みで、「発送済み」の注文詳細画面を開いている、**操作** ステータスを「配送完了」に変更する、**結果** ステータスが「配送完了」に更新される
5. **前提** 管理者としてログイン済みで、「配送完了」の注文詳細画面を開いている、**操作** ステータス変更を試みる、**結果** 変更不可であり、遷移先の選択肢が表示されない
6. **前提** 管理者としてログイン済みで、「キャンセル」の注文詳細画面を開いている、**操作** ステータス変更を試みる、**結果** 変更不可であり、遷移先の選択肢が表示されない
7. **前提** 管理者としてログイン済みで、「発送済み」の注文詳細画面を開いている、**操作** ステータスを「キャンセル」に変更しようとする、**結果** 遷移不可であり、「キャンセル」は選択肢に含まれない

---

### エッジケース

- カートに商品を追加した後、別のタブで同じ商品が在庫切れになった場合の注文確定時の処理 → 注文確定時にカート内容の整合性チェックを行い、在庫不足がある場合はエラーメッセージを表示する
- 注文確定ボタンを連続クリックした場合の二重注文防止 → 注文確定ボタンクリック後はローディング状態にし、二重送信を防止する
- 注文確定処理中にネットワークエラーが発生した場合 → エラーメッセージを表示し、再試行を促す
- 注文一覧でページネーション遷移中に新しい注文が追加された場合 → 次回のページ読み込み時に反映される（リアルタイム更新は不要）
- 管理者が存在しない注文IDで詳細画面にアクセスした場合 → 404エラーを表示する

## 要件

### 機能要件

- **FR-001**: 購入者は、カートに商品がある状態でチェックアウト画面を開き、商品一覧（商品名・単価・数量・小計）と合計金額を確認できなければならない（MUST）
- **FR-002**: 購入者は、チェックアウト画面から「注文を確定」ボタンで注文を作成できなければならない（MUST）
- **FR-003**: 注文確定後、注文完了画面に注文IDとお礼メッセージを表示しなければならない（MUST）
- **FR-004**: 注文確定後、カートは自動的にクリアされなければならない（MUST）
- **FR-005**: カートが空の場合、注文確定ボタンは非表示にしなければならない（MUST）
- **FR-006**: 注文確定時、二重送信を防止しなければならない（MUST）
- **FR-007**: 注文金額にはカートの税抜小計を使用しなければならない（MUST）
- **FR-008**: 購入者は、自分の注文のみ一覧で確認できなければならない（MUST）
- **FR-009**: 注文一覧には注文ID先頭8桁・注文日・商品数・合計金額・ステータスを表示しなければならない（MUST）
- **FR-010**: 注文一覧は20件/ページのページネーションで表示しなければならない（MUST）
- **FR-011**: 購入者は、注文詳細画面で注文ID・注文日時・ステータス・商品一覧・合計金額を確認できなければならない（MUST）
- **FR-012**: 購入者は、他人の注文にアクセスできてはならない（MUST NOT）
- **FR-013**: 管理者は、全ユーザーの注文を一覧で確認できなければならない（MUST）
- **FR-014**: 管理者は、ステータスで注文一覧を絞り込めなければならない（MUST）
- **FR-015**: 管理者は、注文のステータスをステータス遷移ルールに従って更新できなければならない（MUST）
- **FR-016**: ステータス遷移ルールに違反する変更は拒否しなければならない（MUST）
- **FR-017**: 購入者は、管理者の注文管理ページにアクセスできてはならない（MUST NOT）
- **FR-018**: 注文作成時に新しい注文IDが一意に生成されなければならない（MUST）
- **FR-019**: 注文作成時のステータスは「保留中」（pending）でなければならない（MUST）
- **FR-020**: チェックアウト・注文履歴・注文詳細への操作は購入者（buyer）ログインが必須であり、未ログイン時はログインページにリダイレクトしなければならない（MUST）
- **FR-021**: 注文管理への操作は管理者（admin）ログインが必須でなければならない（MUST）

### 主要エンティティ

- **注文（Order）**: 注文を表す。注文ID（一意識別子）、注文者ユーザーID、注文日時、ステータス（pending / confirmed / shipped / delivered / cancelled）、注文明細のリスト、合計金額を持つ
- **注文明細（OrderItem）**: 注文に含まれる個々の商品を表す。商品ID、商品名、単価、数量、小計を持つ。注文に紐づく
- **ステータス遷移ルール**: 注文ステータスの許可された遷移を定義する。保留中→確認済み/キャンセル、確認済み→発送済み/キャンセル、発送済み→配送完了。配送完了・キャンセルは最終状態

## 成功基準

### 測定可能な成果

- **SC-001**: 購入者がカートに商品がある状態から注文完了画面の表示まで、3クリック以内（チェックアウト画面遷移 → 注文確定 → 完了画面表示）で完了できる
- **SC-002**: 注文確定後、カートが空になっていることを購入者が即座に確認できる
- **SC-003**: 購入者が注文履歴ページから任意の注文詳細を2クリック以内で確認できる
- **SC-004**: 管理者がステータスフィルタを使用して、目的の注文を5秒以内に見つけられる
- **SC-005**: ステータス遷移ルールに違反する操作が100%拒否される
- **SC-006**: 購入者が他人の注文情報にアクセスできないことが100%保証される
- **SC-007**: 注文一覧ページの初回読み込みが3秒以内に完了する

## 前提条件

- カタログ機能およびカート管理機能が実装済みであること
- セッション認証が実装済みで、購入者（buyer）と管理者（admin）のロール判定が可能であること
- カートの商品データ（商品名・単価・数量）がカート機能から取得可能であること
- 消費税は一律10%（注文金額にはカートの税抜小計を使用）
- 注文のキャンセルは管理者のみが実行可能
- 注文データの永続化はインメモリとし、データベース永続化は将来フェーズとする
- 未ログイン時はログインページにリダイレクトし、ログイン後は元のページに戻る

## スコープ外

- 決済処理（クレジットカード、銀行振込など）
- 配送先住所の入力・管理
- 注文のメール通知
- 注文の返品・返金処理
- 購入者による注文キャンセル
- 注文の検索機能（管理者のステータスフィルタを除く）
- 注文の印刷・PDF出力
